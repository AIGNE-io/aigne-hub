# 版本發行說明

本文件詳細記錄了 AIGNE Hub 的所有重要變更、新功能和錯誤修復。DevOps 和 SRE 團隊可利用此資訊追蹤平台的演進，了解每次更新的影響，並據此安排升級計畫。

## 2025 年 10 月

### 新功能

*   **支援影片生成**：平台的核心能力已擴展，以支援影片生成模型。`ModelCalls` 資料表中新增了 `video` 類型，用於追蹤這些請求。
*   **媒體時長分析**：為加強多媒體內容的使用情況追蹤，`Usages` 資料表中新增了 `mediaDuration` 欄位。這使得對音訊和影片生成的成本與消耗分析更加精確。
*   **請求追蹤**：所有模型呼叫中新增了 `traceId` 欄位。此識別碼可讓請求在不同系統元件之間被一致地追蹤，從而提升可觀測性，並簡化偵錯與效能分析。

## 2025 年 9 月

### 新功能

*   **用於負載平衡的憑證權重**：AI 供應商憑證中新增了 `weight` 屬性。此功能支援加權負載平衡，讓管理員能根據預設的優先級將 API 流量分配到多個金鑰，例如將更多請求路由至額度更高或成本更低的憑證。
*   **憑證錯誤追蹤**：`AiCredentials` 資料表中新增了 `error` 欄位。這讓系統能夠儲存與特定憑證相關的最後一個已知錯誤，從而改善診斷並更容易識別和解決供應商 API 金鑰的問題。

## 2025 年 8 月

### 新功能

*   **模型健康監控**：引入了一套新系統，用於追蹤個別 AI 模型的運行狀態。`AiModelStatuses` 資料表現在會記錄每個模型的健康狀況與可用性，從而更深入地了解供應商端的問題，並提高整體系統的可靠性。
