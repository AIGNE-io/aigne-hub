# リリースノート

このドキュメントでは、AIGNE Hub のすべての注目すべき変更、新機能、バグ修正の詳細な記録を提供します。DevOps および SRE チームは、この情報を利用してプラットフォームの進化を追跡し、各アップデートの影響を理解し、それに応じてアップグレードを計画できます。

## 2025年10月

### 新機能

*   **動画生成のサポート**：プラットフォームの中核機能が拡張され、動画生成モデルをサポートするようになりました。これらのリクエストを追跡するために、`ModelCalls` テーブルに新しい `video` タイプが追加されました。
*   **メディア持続時間分析**：マルチメディアコンテンツの使用状況追跡を強化するため、`Usages` テーブルに `mediaDuration` フィールドが追加されました。これにより、音声および動画生成のコストと消費量をより正確に分析できます。
*   **リクエストトレーシング**：すべてのモデルコールに `traceId` フィールドが追加されました。この識別子により、異なるシステムコンポーネント間でリクエストを一貫して追跡できるようになり、デバッグとパフォーマンス分析が簡素化され、可観測性が向上します。

## 2025年9月

### 新機能

*   **負荷分散のための認証情報重み付け**：AI プロバイダーの認証情報に `weight` 属性が追加されました。この機能により、重み付けされた負荷分散が可能になり、管理者は事前定義された優先順位に基づいて複数のキーに API トラフィックを分散できます。例えば、より高い制限や低コストの認証情報に多くのリクエストをルーティングするなどです。
*   **認証情報のエラー追跡**：`AiCredentials` テーブルに `error` フィールドが追加されました。これにより、システムは特定の認証情報に関連する最後に確認されたエラーを保存でき、診断が改善され、プロバイダーの API キーに関する問題の特定と解決が容易になります。

## 2025年8月

### 新機能

*   **モデルのヘルスモニタリング**：個々の AI モデルの運用状況を追跡する新しいシステムが導入されました。`AiModelStatuses` テーブルは各モデルのヘルスと可用性を記録し、プロバイダー側の問題に対するより良い洞察を提供し、システム全体の信頼性を向上させます。
